APRS ALERT

🚨 APRS ALERT

Single CSV field. Maximum 20 entries. Example: SQ3NIK-9, SP3YOR, SP3KON* (trailing * = any SSID)
When disabled, the watch list remains saved but alerts are not reported.
0 = alert on every receive. Range: 0-86400 seconds.
Range: 1-60 seconds. Default: 5 seconds.
Applies only to APRS stations with SSID -7 or -9 (e.g. SP3KON-9, SQ3NIK-7).
When enabled, APRS ALERT will also detect stations marked with WX symbol and show an alert with decoded weather data.
Alert triggers for mobile stations at or below this distance.
Turns APRS ALERT LED blinking on or off. You can solder a 3V buzzer.
Range: 100-60000 ms. Default: 5000 ms.
Range: 50-5000 ms. Default: 500 ms.

WEATHER ALERT

⚡ ALERT POGODOWY / Weather Alert

Shows alert when a possible thunderstorm appears in the next 6 hours forecast.
Shows alert when strong wind is detected in the forecast up to 6 hours ahead.
Set wind speed threshold for alerts (range 1.0-60.0 m/s).

ESP32-HAM-CLOCK Configuration

📶 WiFi Connection

WiFi network name the device should connect to
WiFi password (leave empty for open networks)
Secondary SSID used when the primary is unavailable
Password for the fallback network (leave empty for open network)

🔌 UART Serial Port

Default is 9600. Use the same baud rate on both sides.
Typical value is 8 for GPS and HC-05 modules.
UART2 pin mapping: RX = IO27, TX = IO22 (ESP32).
GPS: Inactive FIX: No
BT Module: Inactive

📻 DX Cluster

DX Cluster server address (e.g. dxc.metronet.pl or dxspots.com)
DX Cluster server port (usually 23 for Telnet)
Hides administrative announcements from the cluster
Hides WWV bulletins (atomic clocks)
Hides WCY bulletins (atomic clocks)
Enables advanced spot filters (requires commands below)
Filter commands for CC-Cluster. Examples: set/filter k,ve/pass (K and VE only), set/filter dxbm/pass hf (HF only), set/filter dxbm/reject 6-cw,6-ssb,6-fm (reject 6m)

🏕️ POTA Cluster

Address used by the backend to fetch POTA spots (HTTP JSON)

🛰️ HAMALERT

Optional. Default: hamalert.org
Optional. Default: 7300
Optional. HamAlert account login for Telnet authentication
Optional. HAMALERT is disabled when login/password are empty

📡 Your Station

Your callsign used to log in to DX Cluster
Maidenhead locator (4-6 chars) used to calculate spot distances
Hour offset from UTC (e.g. Poland: 1)
Latitude in decimal format (like Google Maps). Example: 52.4064 for Poznan
Longitude in decimal format (like Google Maps). Example: 16.9252 for Poznan
When enabled and GPS has FIX, all services will use GPS module coordinates instead of manual station coordinates.

🔍 QRZ.com

QRZ.com API username (optional, for extra callsign info)
QRZ.com API password (optional)
QRZ.com connection status (auto-updated)

🌤️ OpenWeather

API key from openweathermap.org (required to show weather and forecast on TFT screens)

📡 KiwiSDR

Enter the KiwiSDR address you want to listen to stations from, e.g. http://okno.ddns.net:8073/ List of available KiwiSDR: http://kiwisdr.com/.public/

📻 APRS-IS

APRS-IS server address (e.g. rotate.aprs2.net)
APRS-IS server port (default 14580)
Your callsign used to log in to APRS-IS
SSID appended to callsign for APRS beacon TX. 0 = no suffix.
APRS-IS passcode (generate at aprs.org)
APRS station filter radius in kilometers (0-50 km). Filter and all distance calculations use GPS coordinates from the "Your Station" section (format xx.xxxxxx)
Periodically sends your position (TCPIP*). Requires configured coordinates.
First char is table (e.g. / or \), second is symbol (e.g. -). Used in the beacon.
Max 43 ASCII chars. Used on every beacon; the first and every fifth beacon uses the ESP32-HAM-CLOCK project link.
Gap between consecutive beacons after the first (which goes at 1 minute). Range 1-180 min.

📁 Settings Backup

Creates a backup of the current form settings and saves it to your computer.
Loads settings from a backup file. After import, you can save them to the device immediately.
